१. XML फाईल नक्की काय असते?
हे HTML सारखेच दिसते (दोघांमध्ये <tags> असतात), पण याचा उद्देश वेगळा आहे.

HTML माहिती कशी दाखवायची (Design) हे ठरवते, तर XML माहिती काय आहे (Data) हे साठवते.

यातील टॅग्स तुम्ही स्वतःच्या गरजेनुसार ठरवू शकता (म्हणून याला Extensible म्हणतात).

२. XML फाईलचे उदाहरण:
समजा तुम्हाला एखाद्या विद्यार्थ्याची माहिती साठवायची असेल, तर XML मध्ये ती अशी दिसेल:

XML

<student>
    <name>Rahul Patil</name>
    <age>18</age>
    <course>DevOps</course>
</student>


. DevOps आणि Java मध्ये XML चा वापर कुठे होतो?
तुम्ही Java Spring Boot आणि Jenkins शिकणार आहात, तिथे XML चे खूप महत्त्व आहे:

Maven (pom.xml): Java प्रोजेक्टमध्ये कोणकोणत्या लायब्ररीज (Dependencies) वापरायच्या आहेत, हे सांगण्यासाठी pom.xml फाईल वापरली जाते. ही तुमच्या प्रोजेक्टची सर्वात महत्त्वाची फाईल असते.

Jenkins: जेनकिन्सचे कॉन्फिगरेशन आणि जॉब्सची माहिती बॅकएंडला XML फॉरमॅटमध्ये साठवली जाते.

Android Development: अँड्रॉइड ॲप्सचे डिझाइन XML मध्ये केले जाते.

Configuration: जुन्या काळात स्प्रिंग फ्रेमवर्कचे सर्व कॉन्फिगरेशन XML मध्ये असायचे (आता प्रामुख्याने 'Annotations' वापरतात).

५. तुम्हाला काय माहित असणे गरजेचे आहे?
DevOps इंजिनिअर म्हणून तुम्हाला पूर्ण XML कोड लिहायची गरज नसते, पण तुम्हाला ती फाईल वाचता (Read) आली पाहिजे आणि त्यात थोडे बदल (Edit) करता आले पाहिजेत (उदा. pom.xml मध्ये व्हर्जन बदलणे).
